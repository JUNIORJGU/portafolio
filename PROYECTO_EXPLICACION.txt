1. ¿QUÉ ES ESTE PROYECTO?
================================================================================

"My Coding Showcase" es una aplicación web moderna y responsiva que funciona como un portafolio personal. Se trata de un sitio de una sola página donde se presenta:

  - Datos personales del desarrollador
  - Habilidades técnicas y experiencia profesional
  - Proyectos realizados
  - Sección de contacto
  - Soporte multilingüe (español, inglés, portugués y ruso)

El propósito del proyecto es ofrecer a un desarrollador una plataforma profesional y atractiva para mostrar su trabajo.


2. ¿EN QUÉ ESTÁ BASADO?
================================================================================

Este proyecto está construido sobre un stack moderno de desarrollo web:

Herramienta de construcción (build tool):
  - Vite: entorno de compilación rápido y liviano (suele sustituir a Webpack)
    Proporciona: servidor de desarrollo veloz, recarga automática y empaquetado eficiente

FRAMEWORK Y LENGUAJE:
  - React 18: Biblioteca para construir interfaces de usuario con componentes
  - TypeScript: Lenguaje tipado que se compila a JavaScript (da mayor seguridad)
  - JSX/TSX: Sintaxis que mezcla HTML con JavaScript

ESTILIZADO Y DISEÑO:
  - Tailwind CSS: Framework de CSS utilitario (crea estilos sin escribir CSS puro)
  - shadcn/ui: Biblioteca de componentes React pre-construidos y customizables
  - Framer Motion: Biblioteca para animaciones y transiciones suaves

ROUTING Y NAVEGACIÓN:
  - React Router: biblioteca para la navegación entre vistas sin recargar la página
  - BrowserRouter: componente que habilita el enrutamiento en el navegador

ESTADO Y DATOS:
  - React Context API: Sistema nativo de React para compartir estado global
  - React Query (@tanstack/react-query): gestión de datos remotos y caché para peticiones al servidor

COMPONENTES Y LIBRERÍAS:
  - Radix UI: Componentes accesibles de bajo nivel (utilizados para toasts y tooltips)
  - Lucide React: Iconos SVG customizables
  - Sonner: Notificaciones toast elegantes

UTILIDADES:
  - clsx: Combina clases CSS condicionalmente
  - tailwind-merge: Resuelve conflictos en clases de Tailwind
  - next-themes: Manejo de temas (light/dark)

4. ¿PARA QUÉ SIRVE CADA ARCHIVO IMPORTANTE?
================================================================================

CONFIGURACIÓN Y ENTRADA
─────────────────────────────────────────────────────────────

index.html
  - Es el archivo HTML base que el navegador carga
  - Define el título de la página, meta tags (SEO), fuentes
  - Contiene <div id="root"></div> donde React monta la aplicación
  - Carga el archivo main.tsx que inicializa React

main.tsx
  - Archivo JavaScript/TypeScript que se ejecuta primero
  - Crea la raíz de React y renderiza el componente App
  - Es el "punto de entrada" de toda la aplicación

App.tsx
  - Componente raíz de toda la aplicación
  - Configura los "proveedores" que necesita toda la app:
    * QueryClientProvider: Habilita React Query
    * TooltipProvider: Habilita tooltips en toda la app
    * Toaster/Sonner: Habilita notificaciones
    * BrowserRouter: Habilita enrutamiento
    * Routes: Define qué componente se muestra según la URL

src/pages/Index.tsx
  - Define la estructura de la página principal
  - Envuelve con LanguageProvider (soporte de idiomas)
  - Renderiza los componentes en orden: Navbar, HeroSection, etc.

COMPONENTES DE SECCIÓN
─────────────────────────────────────────────────────────────

HeroSection.tsx
  - Sección principal/portada (contenido visible sin desplazarse)
  - Contiene saludo, descripción y botones de acción
  - Utiliza animaciones de Framer Motion

AboutSection.tsx
  - Sección que cuenta más sobre la persona
  - Incluye logo/avatar
  - Proporciona contexto e información

DevSection.tsx
  - Muestra información del desarrollador
  - Ubicación, rol actual, educación, biografía
  - Información resumida del perfil

SkillsSection.tsx
  - Exhibe habilidades técnicas
  - Agrupa por categorías (Frontend, Backend, etc)
  - Muestra iconos y nombres de tecnologías

ProjectsSection.tsx
  - Galería de proyectos realizados
  - Cada proyecto es una tarjeta con descripción
  - Enlaces a proyectos en vivo o repositorios

ContactSection.tsx
  - Formulario de contacto
  - Validación de datos
  - Enlaces a redes sociales

Navbar.tsx
  - Barra de navegación fija en la parte superior
  - Menú responsive (móviles y escritorio)
  - Selector de idiomas
  - Efecto de desplazamiento (cambia apariencia al scrollear)

CONTEXTOS Y HOOKS
─────────────────────────────────────────────────────────────

LanguageContext.tsx
  - Crea un Context para proporcionar idiomas a toda la app
  - Contiene las traducciones para 4 idiomas
  - Proporciona funciones para cambiar idioma
  - Los componentes usan useLanguage() para acceder a traducciones

  - Hook personalizado que detecta dispositivos móviles
  - Retorna true/false según el ancho de pantalla
  - Útil para mostrar/ocultar elementos según dispositivo

use-toast.ts
  - Hook para mostrar notificaciones tipo "toast"
  - Permite mostrar mensajes de éxito, error, etc
  - Gestiona la cola de notificaciones

CONFIGURACIÓN
─────────────────────────────────────────────────────────────

vite.config.ts
  - Configura Vite (el bundler/servidor de desarrollo)
  - Puerto: 8080
  - Plugins: React SWC (compilador rápido)
  - Alias de importación: @ apunta a src/

tailwind.config.ts
  - Configura Tailwind CSS
  - Define colores personalizados (tema)
  - Modo oscuro habilitado
  - Fuentes personalizadas: JetBrains Mono y Inter

tsconfig.json
  - Configura TypeScript en el proyecto
  - Define rutas de importación (@/*)
  - Configura opciones del compilador

package.json
  - Lista todas las dependencias del proyecto
  - Define scripts para desarrollar (npm run dev)
  - Meta información del proyecto (nombre, versión)

ESTILOS
─────────────────────────────────────────────────────────────

index.css
  - Importa Tailwind CSS (base, components, utilities)
  - Define variables CSS globales (colores, espaciado)
  - Define animaciones personalizadas
  - Setup de fuentes de Google

App.css
  - Estilos específicos del componente App
  - Actualmente vacío o con mínimo contenido


5. FLUJO DE LA APLICACIÓN (CÓMO FUNCIONA)
================================================================================

INICIACIÓN:
1. El navegador carga index.html
2. Se ejecuta main.tsx que:
   - Obtiene el elemento DOM con id="root"
   - Crea una raíz de React en ese elemento
   - Renderiza el componente <App />

RENDERIZADO INICIAL:
3. App.tsx se renderiza y configura:
   - QueryClientProvider para React Query
   - BrowserRouter para enrutamiento
   - Routes que mapean URLs a componentes
   - Por defecto, "/" muestra el componente Index

4. Index.tsx se renderiza y:
   - Proporciona LanguageProvider (context de idiomas)
   - Renderiza Navbar (barra de navegación)
   - Renderiza todas las secciones en orden vertical

INTERACCIÓN DEL USUARIO:
5. Usuario ve la página con:
   - Navbar en la parte superior
   - Las secciones scroll hacia abajo
   - Puede cambiar idioma en el navbar
   - Puede ver sus proyectos, habilidades
   - Puede contactar a través del formulario

CAMBIO DE IDIOMA:
6. Usuario selecciona idioma en Navbar:
   - Actualiza el Context de LanguageContext
   - Todos los componentes que usan useLanguage() se re-renderizan
   - El contenido se traduce automáticamente

RESPUESTA RESPONSIVA:
7. Tailwind CSS hace que la app se adapte:
   - En móvil: layout vertical, menú hamburguesa
   - En desktop: layout horizontal, menú completo
   - Breakpoints automáticos


6. FLUJO DE DATOS
================================================================================

GLOBAL STATE (Contexto):
- LanguageContext: mantiene el idioma actual y funciones para traducir
  Acceso: cualquier componente que importe useLanguage()

LOCAL STATE:
- Cada componente puede tener su propio estado con useState()
- Ejemplo: Navbar tiene estado para menú abierto/cerrado

VALIDACIÓN:

ANIMACIONES:
- Framer Motion: anima elementos al renderizar/interactuar
- Transiciones CSS: Tailwind proporciona clases para animaciones

9. CÓMO DESARROLLAR CON ESTE PROYECTO
================================================================================

INSTALACIÓN INICIAL:
$ npm install                    # Instala todas las dependencias

DESARROLLO:
$ npm run dev                    # Inicia servidor de desarrollo en puerto 8080
                                 # Con hot reload (actualiza al guardar archivos)

CHEQUEO DE CÓDIGO:
$ npm run lint                   # Ejecuta ESLint para encontrar problemas

CONSTRUCCIÓN PARA PRODUCCIÓN:
$ npm run build                  # Compila para producción a carpeta 'dist/'
$ npm run preview                # Previsualiza build de producción localmente

CONSTRUCCIÓN CON MODO DEV:
$ npm run build:dev              # Compila pero con modo desarrollo


10. DÓNDE EDITAR QUÉ
================================================================================

Para cambiar el contenido:
- Editar componentes en src/components/ (título, descripción, etc)

Para cambiar traducciones:
- Editar LanguageContext.tsx en src/contexts/

Para añadir nueva sección:
- Crear nuevo archivo en src/components/
- Importarlo en src/pages/Index.tsx
- Renderizarlo en el JSX

Para cambiar colores/tema:
- Editar src/index.css (variables CSS)
- O editar tailwind.config.ts (tema Tailwind)

Para cambiar fuentes:
- Editar tailwind.config.ts (fontFamily)

Para añadir nueva página:
- Crear archivo en src/pages/
- Importarlo en App.tsx
- Añadir ruta en <Routes> de App.tsx

Para cambiar estilos globales:
- Editar src/index.css
